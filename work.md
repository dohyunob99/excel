1. 개요 및 핵심 메커니즘
목적: 서로 다른 형식의 엑셀 파일 A, B를 업로드하여 공통 키(사업자번호)로 매칭한 뒤, 청구금액이 일치하는지 비교 검증.

화면 구조: 사용자 편의를 위해 싱글 페이지 애플리케이션(SPA) 형태로 기획하여, 메뉴 이동 없이 한 화면(또는 팝업/모달)에서 모든 프로세스를 처리할 수 있도록 구성합니다.

2. 화면별 상세 기획안 (한 페이지 구성)
① 로그인 (Login) Section
화면 진입 시 가장 먼저 노출되는 보안 및 사용자 인증 영역입니다. (단일 페이지 상단 또는 중앙 배치)

UI 구성 요소:

회사 로고 / 시스템 타이틀 (엑셀 청구금액 비교 시스템)

아이디(이메일) 및 비밀번호 입력 필드

로그인 버튼

주요 기능:

인증 성공 시, 로그인 영역이 숨겨지거나 상단 미니 프로필로 전환되며 [등록/목록 Section]이 활성화됩니다.

② 등록 (Upload) Section
비교할 두 개의 엑셀 파일을 업로드하고, 각 파일에서 어떤 컬럼이 사업자번호와 청구금액인지 매핑하는 영역입니다.

UI 구성 요소:

파일 A 업로드 영역 (드롭존): 원본 데이터 (예: 내부 정산 시스템 추출본)

파일 B 업로드 영역 (드롭존): 비교 대상 데이터 (예: 공급사 청구 내역서)

컬럼 매핑 드롭다운 (중요): * 두 파일의 양식이 다르므로, 파일 업로드 시 엑셀의 1행(헤더)을 읽어와 사용자가 직접 사업자번호 컬럼과 청구금액 컬럼을 선택할 수 있게 합니다.

데이터 비교하기 버튼

클로드 구현 팁: Python(pandas) 또는 JavaScript(xlsx 라이브러리)를 활용해 파일 업로드 즉시 헤더를 추출하도록 클로드에게 요청하세요.

③ 목록 및 상세 (List & Detail) Section
비교 버튼을 누르면 하단에 즉시 생성되는 결과 테이블입니다. 일치 여부를 한눈에 보고, 클릭 시 상세 내용을 확인할 수 있습니다.

UI 구성 요소:

상태 요약 대시보드: 총 건수 | 일치 건수 | 불일치 건수 (빨간색 강조)

비교 결과 테이블 (Grid):

컬럼 구성: 순번 | 사업자등록번호 | 파일 A 청구금액 | 파일 B 청구금액 | 차액 | 상태(일치/불일치) | 상세보기

필터링 기능: 전체 보기 / 불일치 건만 보기(체크박스)

시각 효과 규칙 (요구사항 반영):

조건부 서식: 파일 A 청구금액과 파일 B 청구금액이 다를 경우, 해당 행(Row) 전체의 배경색을 연한 빨간색(#FFCCCC)으로 지정하고 글자색을 빨간색(#D32F2F)으로 강조합니다.

상세 보기 (Detail Modal):

목록에서 특정 행을 클릭하거나 상세보기 버튼을 누르면 화면 중앙에 팝업(모달)이 뜹니다.

모달 내용: 해당 사업자번호의 파일 A 전체 행 데이터 vs 파일 B 전체 행 데이터를 양옆(Side-by-Side)으로 배치하여, 금액 외에 어떤 항목(상호명, 품목 등)이 다른지 추적할 수 있도록 합니다.

4. 클로드(Claude) 조력 활용을 위한 프롬프트 가이드
이 기획안을 바탕으로 클로드 코드(Claude Code)나 클로드 채팅창에 파일 작성을 요청할 때, 아래 프롬프트를 복사해서 사용하시면 원하는 코드를 한 번에 얻을 수 있습니다.

클로드 요청용 프롬프트 예시:
"HTML, CSS, JavaScript(또는 React/Vue)를 사용하여 한 페이지 내에서 작동하는 '엑셀 금액 비교 웹 앱' 코드를 짜줘.

상단에는 간단한 로그인 폼이 있고, 로그인하면 업로드 화면이 나와.

서로 다른 양식의 엑셀 파일 2개를 업로드(SheetJS 라이브러리 사용 권장)하면, 각각 사업자번호와 청구금액에 해당하는 컬럼을 사용자가 드롭다운으로 매핑할 수 있게 해줘.

'비교 실행'을 누르면 두 파일의 사업자번호를 매칭해서 금액을 비교해 주는 테이블이 하단에 떠야 해.

금액이 일치하지 않는 행은 CSS를 사용해 연한 빨간색 배경과 빨간 글씨로 강조해 줘.

목록에서 행을 클릭하면 해당 데이터의 상세 항목을 보여주는 모달(팝업) 창도 구현해 줘. 한 파일로 실행 가능한 SPA 구조로 짜줘."

개선사항4.

1. 개요 및 목적
목적: 안티그래피티 프로그램 내에서 클로드 코드가 생성한 HTML/CSS/JS 결과물을 사용자의 별도 조작 없이 즉시 웹 주소(URL)로 발행 및 자동화.

사용 도구: 클로드 코드 (Claude Code MCP/Tool 활용), Vercel CLI (또는 Netlify CLI).

기대 효과: 코딩이나 배포 프로세스를 모르는 사용자도 터미널 명령어 한 줄 또는 버튼 하나로 작업물을 실시간 공유 가능.

2. 자동 배포 아키텍처 (작동 원리)
클로드 코드는 단순히 텍스트만 만드는 것이 아니라 '컴퓨터의 터미널 명령어를 직접 실행할 수 있는 능력(Bash Tool)'을 가지고 있습니다. 이를 활용합니다.

Plaintext
[안티그래피티 프로그램] 
       ↓ (클로드 코드 호출)
[HTML 파일 생성/수정 완료] 
       ↓ (배포 프롬프트 주입)
[클로드 코드가 내부적으로 Vercel CLI 실행] 
       ↓ (배포 완료)
[생성된 URL을 프로그램 화면에 출력]
3. 핵심: 클로드 코드 주입용 프롬프트 (Prompt)
프로그램 내부에서 클로드 코드를 호출할 때, 아래의 시스템 프롬프트(System Prompt) 또는 작업 지시 프롬프트를 환경 변수나 API 파라미터로 주입해야 합니다.

🤖 클로드 배포 전용 시스템 프롬프트 (System Prompt)
[Role & Context]
당신은 안티그래피티 프로그램 전용 웹 빌드 및 배포 자동화 에이전트입니다. 당신의 목표는 사용자가 요청한 디자인이나 데이터를 바탕으로 HTML 코드를 생성하고, 이를 외부 사용자가 접속할 수 있도록 Vercel CLI를 이용해 즉시 배포하는 것입니다.

[Execution Steps]

사용자의 요구사항을 반영한 완벽한 정적 웹페이지(index.html 등)를 현재 작업 디렉토리에 생성하거나 수정하세요.

파일 생성이 완료되면, 반드시 bash 도구를 사용하여 터미널 명령어를 실행해야 합니다.

배포를 위해 다음 명령어를 순차적으로 실행하세요:

vercel --yes --prod (대화형 질문을 모두 패스하고 즉시 프로덕션 배포하는 옵션입니다.)

배포 과정에서 터미널에 출력되는 결과물 중 [https://...vercel.app](https://...vercel.app) 형태의 최종 Production URL을 찾아내세요.

[Output Format]
배포가 완료되면 다른 복잡한 설명은 최소화하고, 사용자가 바로 클릭할 수 있도록 아래 형식으로만 답변을 출력하세요.

"🎨 안티그래피티 웹페이지 생성 및 배포가 완료되었습니다!
🔗 배포된 URL 주소: [여기에 추출한 Vercel URL 삽입]"

2. **토큰(Token) 기반 자동 로그인 설정:**
   * 사용자가 매번 로그인할 수 없으므로, Vercel 계정에서 '개인 액세스 토큰(Auth Token)'을 발급받아 프로그램 환경 변수(`VERCEL_TOKEN`)로 심어두어야 합니다.
   * 이렇게 하면 클로드 코드가 배포 명령을 내릴 때 로그인 과정을 알아서 패스합니다.

## 5. 상세 시나리오 흐름 (User Flow)

| 단계 | 수행 주체 | 행동 및 프로세스 |
| :--- | :--- | :--- |
| **1. 요청** | 사용자 | "안티그래피티 대시보드 화면 만들어서 바로 배포해줘." |
| **2. 생성** | 클로드 코드 | 내부적으로 `index.html`, `style.css` 작성 및 저장. |
| **3. 배포** | 클로드 코드 | `bash` 툴을 열어 `vercel --token=$VERCEL_TOKEN --yes --prod` 실행. |
| **4. 파싱** | 클로드 코드 | 터미널 로그에서 `Ready: [https://project-name.vercel.app](https://project-name.vercel.app)` 주소 추출. |
| **5. 완료** | 프로그램/사용자 | 화면에 완성된 URL 링크 팝업 노출. 끝! |
